## Dialogue Specifications

### Language & Tone
- **Language**: Natural Bangla (not overly formal Sadhu Bhasha, not street slang)
- **Register**: Semi-formal — professional but warm
- **Agent addressing**: Always "স্যার" (Sir) or "ম্যাম" (Ma'am)
- **Customer addressing**: Can say "ভাই", "আপু", or just directly speak

### Dialogue Structure Per Conversation
Each conversation must have **20+ dialogues** (alternating Agent/Customer), following this flow:

```
1. Agent: Greeting + company name + ask how to help
2. Customer: State the problem/query
3. Agent: Acknowledge + ask for verification (phone/order number)
4. Customer: Provide information
5. Agent: Confirm identity + ask details about issue
6. Customer: Explain situation in detail
7-14. Agent/Customer: Back-and-forth troubleshooting / information exchange
15-18. Agent: Provide solution / escalate / set expectations
19-20. Agent: Summary + anything else? + closing
20+. Customer: Thank + goodbye
```

### Word Count Rule
Each dialogue turn: **20-30 Bangla words total**. Mix of:
- 2-3 short simple sentences, OR
- 2 medium sentences, OR  
- 1 long + 1 short sentence

## Execution Plan

### Chunking Strategy

Each generation chunk produces **4 conversations** for a **single script**. This keeps output quality high and avoids overwhelming the model.

| Parameter | Value |
|-----------|-------|
| **Conversations per chunk** | 4 |
| **Chunks per script** | ~25 (to reach 100+ conversations) |
| **Total chunks across 20 scripts** | ~500 |

#### Chunk Naming Convention
Each chunk appends to its script file. Chunks are numbered sequentially:
```
Script 01 (bKash):   Chunk 1 → সিনারিও ১-৪,  Chunk 2 → সিনারিও ৫-৮,  ...  Chunk 25 → সিনারিও ৯৭-১০০
Script 02 (GP):      Chunk 1 → সিনারিও ১-৪,  Chunk 2 → সিনারিও ৫-৮,  ...  Chunk 25 → সিনারিও ৯৭-১০০
...
Script 20 (Chaldal): Chunk 1 → সিনারিও ১-৪,  Chunk 2 → সিনারিও ৫-৮,  ...  Chunk 25 → সিনারিও ৯৭-১০০
```

#### Workflow Per Script
1. **Chunk 1**: Create the `.md` file with the H1 header + metadata + সিনারিও ১-৪
2. **Chunks 2-25**: Append সিনারিও ৫-৮, ৯-১২, ... ৯৭-১০০ to the same file
3. After all chunks: Verify conversation count, dialogue count, and formatting

### Markdown Formatting Rules

- **H2 (`##`)** — Each scenario heading (`সিনারিও ১ঃ ...`)
- **Blockquote (`>`)** — Scenario brief/description
- **Bold (`**`)** — Dialogue number + speaker label (`**১. এজেন্টঃ**`)
- **Horizontal rule (`---`)** — Separator between scenarios
- **Trailing double-space** — Line breaks between dialogue turns
